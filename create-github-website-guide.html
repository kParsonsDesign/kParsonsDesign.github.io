How to Create your own Website in GitHub

Want to create a portfolio website, but don't want to deal with your own web hosting? GitHub is a great option to create a web page that you can share with others, especially for those interested in coding and software.

This tutorial will walk you through all the steps to create a personal portfolio page to showcase your work.

Steps to create a website on GitHub

  1. Create a GitHub account
  2. Create a repository
  3. Upload your files to your repository

Create an Account on GitHub

The first thing you need to do is create an account on GitHub.

Go to https://github.com in your favorite web browser. Enter the email address you want associated with your new account in the sign up field and then click the green, "Sign up for GitHub" button.

Next, make sure you enter a strong password.

Last, pick a username for your account. Warning: Choose your username carefully! This will be publicly displayed and will be the URL for your public website.

Create a Repository for your Website

After creating your account you need to create a repository to hold your files. This is where you create the area for your public website.

From your account Overview page, click on the Repositories tab. Or click on the plus symbol at the top right of any GitHub page when you are signed in and select the "New Repository" option.

Naming your new repository

The name for your new repository is extremely important. It must take the format <username>.github.io. Example: kparsonsdesign.github.io. Make sure the user name you type in matches your account user name exactly. If you have a typo or just choose something different you can use the repository, but your page won't be available on the internet.

Add Content to your Repository

The last basic step to complete your website is to add your files to your repository.

Create a simple index.html file on your computer that will be your home page. It should at least have some content in it so you can see it from your web browser after you upload it.

Upload your files to GitHub

After you've created the files to start your website, you need to upload them to GitHub.

The simplest way to upload files is to navigate to your repository page in GitHub. Click on the Repositories tab and select your website repository. Then click the "Add file" button and select "Upload files".

Now you are on the uploading page and can drag files from your file manager to add them into your repository. At least one of the files must be an index.html (or index.md) file to create your public website. When you are done selecting all the files you want to upload you have to "Commit" them to GitHub.

Because GitHub uses version control to keep track of any changes you make to your projects you have to "commit" them. Otherwise they won't save to your repository.

To commit your files to your repository, type in a short description of what this commit does in the present tense. (Example: Uploads index.html) This description lets you know what this particular change did to your code. It might not seem important on this first upload, but after hundreds and thousands you will be glad to know what each change is meant to do. Lastly, select the green "Commit changes" button at the bottom of the page to add your new files to your repository.

Your new website can now be viewed at https://username.github.io.

One thing to note: GitHub pages do not publish instantaneously. After you make a commit, GitHub processes it in the background before it goes live.

If you are not sure if your new changes should be visible or not, navigate to the Settings tab from within your repository. Then click on "Pages" in the left navigation bar. If there is a blue bar at the top of the page, GitHub is still processing and your changes are not yet live. Give it a minute.
